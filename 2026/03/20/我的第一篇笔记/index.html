我的第一篇笔记

一、本节要点

  • 红黑树 = 二叉查找树 + 颜色约束 + 旋转恢复
  • 五条规则决定了黑高一致性，使最坏高度 O(log n)
  • 插入/删除后的“颜色修复+旋转”是核心操作

二、重点高光

  • 红色节点不连续（避免局部链式增长）
  • 从任意节点到叶子路径黑色节点数相同（黑高）
  • 根节点始终黑色，完成平衡后保持不变

高光摘要：

  • 最长路径 ≤ 2 × 最短路径
  • 查找/插入/删除均为 O(log n)
  • 常用于Java TreeMap、C++ std::map、Linux 内核 rbtree

三、详细说明与证明

3.1 黑高性质推导（关键证明过程）

设一棵红黑树中任一节点的黑高为 bh。

  • 基础：空节点（NIL）bh = 0。
  • 若节点为黑，则其任一子树的黑高为 bh - 1；若为红节点，子树黑高为 bh。

从根到任一路径必须包含 bh 个黑节点，且在两个分支上都是一样的。
最长路径包含所有黑节点 + 所有可能的红节点（红节点数量 ≤ 黑节点数量），
所以最长长度 ≤ 2 × 最短长度。

证明：

  • 设某路径黑节点数为 BH，红节点最多为 BH - 1（不能连续红），
  • 则路径长度 ≤ BH + (BH - 1) = 2BH - 1；最短路径长度至少 BH。
  • 因此树高度 h ≤ 2BH - 1。
  • 又由于 BH ≥ log2(n+1)（每个黑节点至少支撑一个完整二叉树），
  • 得 h = O(log n)。

3.2 时间复杂度

  • 查找：O(h) = O(log n)
  • 插入：O(log n) + 修复旋转（最多常数次）= O(log n)
  • 删除：O(log n) + 修复= O(log n)

3.3 插入修复过程带图解释

  1. 新节点插入后染红（红新生）
  2. 若父黑：结束。若父红（违反规则4），进入修复
  3. 叔叔为红：父叔黑，祖父红，向上继续（颜色翻转）
  4. 叔叔为黑：根据三角/直线结构做旋转，并置换颜色

图示假设：左边子树插入导致祖父节点平衡被破坏时的 LL/RR/LR/RL 情况。

3.4 删除修复过程带图解释

  1. 目标节点实质删除或换位后，可能产生“双重黑”（黑高度少1）
  2. 兄弟为红：旋转后变为兄弟黑，继续统一处理
  3. 兄弟为黑且有红子：旋转+颜色调整，直接消除双重黑
  4. 兄弟为黑且无红子：兄弟变红，父节点递归处理
