学习笔记

我的第一篇笔记

一、本节要点

  • 红黑树是一种自平衡二叉查找树
  • 通过“颜色”和“旋转”保证高度对数级
  • 插入、删除、查找的时间复杂度都是 O(log n)

二、详细内容

红黑树可以把它想象成“黑白棋盘上的有序家谱树”，每个节点都有黑色或红色。它不仅记录了左右子树大小关系（左小右大），还附带颜色信息来控制高度。

2.1 红黑树的五条规则

  1. 每个节点要么红要么黑。
  2. 根节点必须是黑色（实现中插入后再调整）。
  3. 叶子节点（NIL 空节点）是黑色。
  4. 红色节点的两个子节点都必须是黑色（不能连续两个红）。
  5. 从任意节点到其所有叶子节点的路径上，黑色节点数量相同。

这五条规则的直观含义：

  • 红色节点当“借用节点”，不能连着两个红；
  • 黑色节点做“高度支撑”，保证所有路径高度接近。

2.2 为什么红黑树能自平衡？

把红色节点想成“可选额外高度”，黑色节点是“必须高度”。任何路径黑色节点一样多，最长路径最多是最短路径的两倍，因此高度 O(log n)。

2.3 插入时的“调色+旋转”流程

  1. 新节点当红节点插入，避免影响黑高。
  2. 如果父节点是黑，完成；如果父节点是红，则需要修复（4 条规则违规）。
  3. 常见修复：
    • 父、叔均红：父叔变黑，祖父变红，继续向上处理（颜色翻转）
    • 父红叔黑：旋转（左旋/右旋）+颜色转换，重构局部平衡

2.4 删除关键点

删除节点会改变黑高，常用策略是用后继替换，然后对“黑高减少”的路径做“借黑”操作：

  • 兄弟节点黑且有红子：旋转+变色，补黑
  • 兄弟节点黑且无红子：兄弟变红，父降黑继续向上
  • 兄弟节点红：旋转转换为兄弟黑再继续处理

2.5 重点回顾

  • 核心是“五条规则”，特别是“红不连”和“路径黑高一致”。
  • 插入删除复杂，查找简单；但整体复杂度保持 O(log n)。
  • 工程中广泛用于 Java TreeMap、C++ std::map、Linux Kernel rbtree 等。
